# Drone Libre Project Charter

## Mission

Drone Libre is a preservation, repair, and modernization effort for small unmanned aircraft systems, with emphasis on technician-friendly documentation, open interfaces, recoverable firmware practices, and modular hardware.

## Core Idea

Useful aircraft should not become disposable simply because a proprietary subsystem fails, support ends, or documentation is unavailable.

The project explores how older drones can be studied, repaired, documented, and extended using open engineering practices.

## Primary Objectives

- Preserve and document older drone hardware.
- Build repeatable diagnostic and repair procedures.
- Develop portable field tools for firmware backup, recovery, logging, and subsystem testing.
- Explore companion electronics for Remote ID, telemetry, and field diagnostics.
- Study open flight-controller and communication architectures.
- Create documentation that can support portfolio, capstone, and future product-development work.

## Project Values

- Repair before replacement.
- Document before modifying.
- Measure before guessing.
- Build adapters before irreversible hacks.
- Separate confirmed data from assumptions.
- Make the work understandable to another engineer.

## Initial Focus Areas

- DJI Mini-class preservation and research
- Handheld field-service tool development
- Remote ID and companion-board concepts
- MAVLink / OpenDroneID / ArduPilot / PX4 study paths
- Hardware mapping and firmware backup procedures

## Non-Goals

- This is not a bypass project.
- This is not a black-box modification guide.
- This is not a claim that any experimental aircraft is flight-ready.
- This is not a replacement for safe test discipline, documentation, or regulatory compliance.

## Working Standard

Every major design idea should eventually answer:

- What problem does it solve?
- What hardware does it affect?
- What data supports the decision?
- Can it be reversed?
- How would another technician inspect it?
- What must be tested before flight?
